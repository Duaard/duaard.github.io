Exploring AI and Machine Learning Through Simulating Shared Consciousness with Personalized Language Models #

As mentioned, I want to learn more AI. And I want to do this by tackling a project with hands-on practical implementation.

Concept Overview #

This project conceptualizes consciousness as a functional 'black box' within humans, processing inputs such as environmental stimuli, emotions, and psychological states to produce discernible actions. We propose an "Integrated Consciousness Framework" (ICF), where consciousness can be extended and integrated with non-conscious entities, enabling the control of their actions akin to those of a conscious being.

Core Principle #

In ICF, an 'Integrator' extends their consciousness to a non-conscious entity, such as a character in a game or novel. This extension enables the Integrator to define and drive the actions of these entities, effectively creating a bridge between conscious intent and non-conscious execution.

Transient Nature of Integration #

The integrated consciousness persists only as long as the Integrator actively engages with the non-conscious entity. For example, a novelist imbues a character with purpose and action through writing. Similarly, a gamer instills consciousness into a game character by controlling it. The integration is suspended when the Integrator disengages, pauses, or ceases interaction, reverting the entity to a non-conscious state.

Role of Large Language Models (LLMs) #

LLMs serve as facilitators in this framework, automating the integration process. By processing a wide array of inputs and producing coherent outputs, LLMs can maintain a constant state of integrated consciousness for created characters or entities. This system ensures that these entities possess an ongoing, albeit artificial, consciousness as long as they are under the LLM's operation.

Objectives #

1. Learning Through Development and Experimentation #

  • Goal: Utilize the project as a hands-on learning experience in AI and machine learning, focusing on the hypothesis that consciousness can be shared with non-conscious entities.
  • Method: Implement and experiment with various AI techniques and models, using the project as a practical testbed for learning.

2. Data Collection and Personalized Model Training #

  • Focus: Gather a diverse set of scenarios and responses to train a personalized language model (LLM).
  • Process: Use these responses to fine-tune the LLM, aiming to simulate decision-making processes similar to those of a human.

3. Simulation Integration and Response Analysis #

  • Aim: Integrate the trained LLM into a simulated environment (such as a game) and analyze its decision-making capabilities.
  • Strategy: Observe how the LLM's responses in the simulation align with expected human decisions, refining the model based on these insights.

4. Ethical Considerations and Responsible AI Use #

  • Responsibility: Address ethical concerns related to AI, particularly in simulating consciousness and decision-making.
  • Approach: Ensure responsible and private use of data, and reflect on the broader ethical implications of the project.

5. Documenting the Learning Journey #

  • Commitment: Maintain a detailed development log (devlog) throughout the project to document challenges, breakthroughs, and learning points.
  • Output: Share this documentation on the blog to provide insights into the project's progress and learnings.

7. Success Metrics: Knowledge Acquisition and Practical Application #

  • Evaluation: Measure the project's success based on the depth and breadth of AI and machine learning knowledge acquired, and the effectiveness of its practical application in the simulated environment.

Steps #

  1. Collect Data and Training Set: Generate a variety of scenarios using GPT-4 to create a realistic word-based simulation. Respond to these scenarios personally to create a training set.
  2. Fine-Tune the LLM Model: Adjust the LLM to generate responses that mimic the human's (my) answers. Cross-check these with actual human data to improve performance.
  3. Simulation Integration: Choose or create a game and develop a system (possibly NLU) to translate dialog into real actions in this world. This system should also translate in-game events into dialog that the LLM can process, potentially using another fine-tuned LLM for translation.
  4. Non-Linear Interaction with Simulation: Note that the simulation doesn’t need to be real-time. This flexibility allows for reduced complexity and non-linear interaction with the simulated world.
  5. Project Outcome: The end goal is a system that can exist in a "simulated world," with interaction capabilities based on the training data and programming.
